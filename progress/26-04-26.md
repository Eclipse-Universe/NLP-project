### 원인 분석 (우선순위 순)

**1순위: get_input_embeddings 패치의 weight tying 간섭**
EXAONE은 `_tied_weights_keys = {"lm_head.weight": "transformer.wte.weight"}`로
lm_head와 embed(wte) 가중치가 묶여있음. PEFT 호환성을 위해 클래스에 직접 패치한
`get_input_embeddings = lambda self: _embed_module` 이 weight tying 메커니즘을 혼란시켰을 가능성.
→ 학습 중 lm_head 가중치 정렬이 깨져 생성 품질 저하

**2순위: 추론 중 greedy fallback 발생**
beam search 중 RuntimeError → 일부 샘플이 greedy로 생성됨.
몇 건이 fallback됐는지 알 수 없으나 ROUGE에 직접 영향

**3순위: device_map 변경 부작용**
`device_map='auto'` → `device_map={'': device}` 변경이 accelerate의
메모리 최적화 동작을 바꿨을 가능성 (분산 처리 비활성화)

**4순위: Qwen2.5-7B-Instruct가 instruction following에 더 최적화**
EXAONE이 한국어 언어 능력은 우수하지만 task 지시 따르기는 Qwen이 더 강할 수 있음
